# Gugu Agent v0.1.18

本次版本聚焦长会话可见性、Gugu Managed 默认体验收敛，以及 gateway 生产加固收尾。

## 新增

- 聊天输入区新增上下文用量环，空闲时轻量估算当前会话上下文占用，点击可打开上下文详情。
- 当上下文达到高水位且当前会话空闲时，桌面端会自动触发 `/compact`，减少用户在长任务中忙等或撞上下文上限的概率。
- Gateway 增加 MySQL 备份恢复校验脚本，可把备份恢复到 scratch/restore/dryrun/test 库后校验 SHA-256、表计数和基础支付订单一致性。

## 改进

- 默认托管入口收敛为 Gugu Managed；其他厂商入口不再作为默认预设暴露，仍可由用户自行配置。
- 长附件解析任务增加轮询与进度提示，避免大附件处理时界面长时间无反馈。
- Gateway attachment task 保持 provider-neutral 协议，payload 使用服务器本机私有 spool，不把用户附件内容放到 OSS。
- Gateway 文档补充 Redis limiter/circuit、MySQL 备份、生产 flags-off 部署和回滚边界，方便后续 agent 继续接手。

## 修复

- 修复订阅过期后续费激活，再执行“回滚对话”时可能暴露 `Session not found` 原始错误的问题；当该轮尚未写入本地会话记录时，会回退为只回滚当前界面里的对话。
- 修复新开 session 选择项目文件夹后，后台刷新可能把刚选中的项目路径清空，底部又显示“选择项目...”的问题。
- 优化单轮连续自动操作达到 40 轮保护上限时的提示，明确这是当前这一轮自动执行保护，不是订阅额度上限，并提示可继续接着处理。

- Gateway 全量测试已通过。
