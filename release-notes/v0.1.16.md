# Gugu Agent v0.1.16

这是一次针对 v0.1.15 打包失败后的修复发布，重点解决安装后名称、侧车服务启动和托管模型超时体验问题。v0.1.15 未作为稳定包继续发布，建议直接使用 v0.1.16。

## 修复

- Windows 安装后的应用名统一为 `gugu-agent`，避免继续显示旧的 `claude-code-desktop`。
- 修复安装包内侧车程序名称不一致导致的“本地服务启动失败 / spawn server sidecar 找不到文件”问题。
- macOS 应用包、窗口标题、DMG 卷标和侧车二进制名称同步为 `gugu-agent` / `gugu-sidecar`。
- 更新桌面端 Computer Use 的 bundle id，和新的应用标识保持一致。
- gateway 托管模型请求增加上游超时保护。上游长时间不返回首包时会明确返回超时错误，并退还本次点数，减少用户长时间卡住的问题。

## 改进

- 打包脚本会清理旧侧车产物，避免旧二进制混入新包。
- Windows 安装/卸载前会尝试停止新旧侧车进程，降低升级时文件占用的概率。
- 保留对旧 `claude-sidecar` 名称的兼容查找，减少老版本升级过程中的失败风险。

## 验证

- `rtk test bun run lint`
- `rtk test cargo check`
- `rtk test bun test gateway/src/__tests__/gateway.test.ts`
