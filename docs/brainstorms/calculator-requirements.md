# 桌面计算器面板

## Summary

在桌面应用的侧边栏/底部面板中新增一个基础计算器，支持加减乘除、百分比、正负号切换和小数点运算，纯按钮交互，类似手机原生计算器。

---

## Problem Frame

桌面应用用户在聊天或浏览过程中，经常需要快速做一些数值计算（如金额汇总、百分比换算）。当前没有内置计算工具，用户需要切换到外部计算器或手动心算，打断工作流。一个轻量、即时可用的内置计算器可以消除这个摩擦。

---

## Requirements

**运算功能**
- R1. 支持加法、减法、乘法、除法四则运算
- R2. 支持小数点和正负号切换（±）
- R3. 支持百分比运算（%）
- R4. 支持连续运算：完成一次计算后，结果可作为下一次运算的初始值

**交互**
- R5. 所有输入通过按钮点击完成，不需要键盘输入
- R6. 点击数字按钮追加到当前显示的数字末尾
- R7. 点击运算符执行相应的计算或准备下一次运算
- R8. 点击等号（=）显示最终结果
- R9. 点击清除（C/AC）重置计算器到初始状态
- R10. 显示区域实时反映当前输入或计算结果

**面板形态**
- R11. 计算器以侧边栏或底部面板的形式嵌入桌面应用现有布局
- R12. 在不需要时可折叠或切换回其他面板，不影响主聊天区域

**视觉风格**
- R13. 遵循桌面应用现有的 UI 设计语言和组件风格
- R14. 按钮布局采用网格排列，显示区域位于面板顶部

---

## Success Criteria

- 用户可以在桌面应用内完成四则运算和百分比计算，无需切换到外部工具
- 计算器面板与现有 UI 风格一致，不引入视觉割裂感
- 面板可自由切换，不占用聊天区域的固定空间

---

## Scope Boundaries

- 不支持科学计算（三角函数、对数、幂运算等）
- 不提供历史记录功能
- 不支持键盘物理输入
- 不提供变量/记忆存储
- 不支持单位换算、进制转换
- 不做独立弹窗或标签页模式

---

## Key Decisions

- **按钮交互优先**：纯点击操作，不引入键盘事件监听，降低实现复杂度和与现有快捷键系统的冲突风险
- **侧边栏/底部面板嵌入**：沿用桌面应用现有面板切换机制，避免引入新的窗口或路由
- **基础计算器功能集**：与手机原生计算器对齐，覆盖最高频的日常计算场景，不做功能膨胀
