## Implemented in this thread

- Local Gugu Managed proxy now rejects oversized request bodies before forwarding to gateway.
- Large `Write(content)` tool payloads are redacted from API conversation history after local writes, preventing the next model call from carrying the full file content again.
- Gateway `/v1/messages` rejects oversized payloads before consuming credits.
- Gateway `/v1/attachments/parse` rejects oversized JSON, oversized decoded files/images, and unsupported MIME types before consuming credits.
- Gateway has independent in-process queues for message and GLM attachment requests.
- Gateway has single-node token bucket limits for device registration, order creation, message, and attachment parsing.
- Payment notification idempotency is in place for WeChat and Alipay, with unique transaction constraints.
- Maintenance mode can disable new orders and show maintenance copy on buy/checkout pages.
- Maintenance mode now has `GUGU_MAINTENANCE_DISABLE_WRITES=1` for full public write freeze during database cutover/rollback rehearsal. It blocks device registration, activation, messages, attachment parsing, order creation, and payment notifications with `503 MAINTENANCE` before mutating storage.
- Gateway now has separate DeepSeek and GLM in-memory circuit breakers.
- DeepSeek streaming responses now have an idle timeout that closes the stream, records metadata, and refunds the request credit.
- GLM request timeout is configurable through `GUGU_GLM_REQUEST_TIMEOUT_MS`.
- Gateway now emits structured request logs when `GUGU_REQUEST_LOG_ENABLED=1`.
- Gateway admin API now exposes `/admin/api/metrics` with request counters, recent requests, queue snapshots, and circuit snapshots.
- Gateway now has `scripts/backup.ts` and `scripts/restore-check.ts` for SQLite `VACUUM INTO` snapshots and restore verification.
- Gateway deploy templates now include `gugu-gateway-backup.service` and `gugu-gateway-backup.timer` for nightly backup plus restore-check.
- Gateway storage now has an explicit `GatewayStorage` interface; the current SQLite `GatewayStore` implements it and HTTP/admin code depends on the interface.
- Added `gateway/src/__tests__/storage-contract.test.ts` as the first storage contract suite before MySQL implementation work.
- Added `gateway/deploy/mysql/schema.sql` as the MySQL 8.4 initial schema draft.
- Added `gateway/scripts/migration-report.ts` and `bun run migration-report` for read-only SQLite migration readiness reports.
